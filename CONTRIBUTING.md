# Contributing to WebCraft AI (SSOC 2026) 🚀

First off, thank you for considering contributing to WebCraft AI! This project is part of the **Social Summer of Code (SSOC) 2026**, and we are excited to have you on board.

Following these guidelines helps communicate that you respect the time of the developers managing and developing this open source project. In return, they should reciprocate that respect in addressing your issue, assessing changes, and helping you finalize your pull requests.

## 🌟 Getting Started

1.  **Fork the Repository**: Click the 'Fork' button at the top right of this page.
2.  **Clone your Fork**:
    ```bash
    git clone https://github.com/YOUR_USERNAME/devhib1.git
    cd devhib1/webcraft-ai
    ```
3.  **Install Dependencies**:
    ```bash
    npm install
    ```
4.  **Set up Environment Variables**: Follow the `.env.example` file to set up your local environment.

## 🛠️ Contribution Process

### 1. Finding an Issue
- Browse the [Issues](https://github.com/Rakshi2609/devhib1/issues) tab.
- Look for issues labeled `ssoc`, `good-first-issue`, or `help-wanted`.
- Comment on the issue to get it assigned to you before you start working.

### 2. Creating a Branch
Create a new branch for your feature or bug fix:
```bash
git checkout -b feature/your-feature-name
```

### 3. Making Changes
- Follow the existing code style.
- Ensure your changes are well-documented.
- If you add a new feature, consider adding a brief explanation in the `README.md` or a new documentation file.

### 4. Committing and Pushing
```bash
git add .
git commit -m "feat: add amazing feature"
git push origin feature/your-feature-name
```

### 5. Opening a Pull Request
- Go to the original repository on GitHub.
- Click "New Pull Request".
- Provide a clear description of your changes and link the issue you addressed.

## 🎨 Style Guide
- **TypeScript**: Use strict typing where possible.
- **Tailwind CSS**: Use utility classes for styling. Avoid inline styles.
- **Next.js**: Use App Router patterns (Server Components by default).

## 🏆 SSOC Rules
- Be respectful to other contributors.
- Do not spam with multiple minor PRs (e.g., just fixing one typo per PR).
- Quality over Quantity! High-quality features and bug fixes earn more recognition.
